## Question 1

You are given two sequence of integers: $(A[1],\dots ,A[n]) \text{ and } (B[1],\dots ,B[m])$. You wish to find the length of the longest increasing common subsequence of the two lists (i.e., the length of the longest increasing subsequence $(A[i_{1}],\dots,A[i_{k}])$ such that there exists a subsequence $(B[j_{1}],\dots,B[j_{k}])$ such that $A[i_{1}]==B[j_{1}]$ and so on up to $A[i_{k}]==B[j_{k}]$. Design a DP tabulation algorithm that returns the length of longest common increasing sub sequence.

**1. Define the subproblem:**
Let $DP[i][j]$ be the length of the longest increasing common subsequence of $A[1],\dots,A[i] \text{ and } B[1],\dots,B[j]$ such that the common subsequence ends exactly with the element $A[i] = B[j]$.

**2. Define and evaluate the base cases:**
The base for the either sequence is empty. Then $DP[i][0] = 0 \text{ for all } 0\leq i\leq n \text{ and } DP[0][j] = 0 \text{ for all } 0\leq j\leq m$.

**3. Establish the recurrence for the tabulation:**
- **Case 1**, if $A[i]\neq B[j]$. Since the subsequence ends must exactly with 
the element $B[j]$, the element $A[i]$ is not the last element of the subsequence. Then, $DP[i][j]=DP[i-1][j]$. Finally, update the max value of the longest increasing subsequence $max(value_{max},DP[i][j])$ when $B[j]<A[i]$.
- **Case 2**, if $A[i] = B[j]$. This shows that the common increasing subsequence between $A \text{ and } B$ could be found. Then $DP[i][j]=1+value_{max}$.
**4. Determine the order of subproblems:**
Since we only calculate the previous values $DP[i-1][j]$ and $DP[i-1][k]$. The outer loop iterates through $i=1 \to n$. The inner loop iterate through $j=1\to m$.

**5. Final form of output:**
The final answer is $Max(DP[n][j])$ where $1\leq j\leq m$.

**6. Pseudocode:**

```java
LICS(A[n],B[m]):
	Init DP[i][0] = 0 for all i < n
	Init DP[0][j] = 0 for all j < m
	for index i from 1 to n:
		max_value = 0
		for index j from 1 to m:
			if A[i] != B[j]:
				DP[i][j] = DP[i-1][j]
				if B[j] < A[i]
					max_value = max(max_value,DP[i][j])
			else
				DP[i][j] = 1 + max_value
				
	result = 0
	for index k from 1 to m:
		result = max(result,DP[n][k])
	return result
```

**7. Runtime analysis:**
- Initial DP for $i \text{ and } j$ takes $O(n)+O(m)$.
- First iteration for $n$ takes $O(n)$.
	- Inner iteration for $m$ takes $O(m)$.
Then, the $LICS(A[n],B[m])$ algorithm takes $O(n)+O(m)+O(n\cdot m)$, which is considered as:

$$
O(n\cdot m)
$$

## Question 2

Team A and Team B are playing a series of matches of their favorite game. Whoever wins $k$ games first will win the series. Each game is independent with Team A having a probability of $p$ to win and Team B having a probability of $1-p$ to win.

Design a DP tabulation algorithm that returns the probability that Team A wins the series.

**1: Define the subproblems**

Let $P(a, b)$ be the probability that Team A wins the series when Team A needs a more wins and Team B needs b more wins.

**2: Define and evaluate the base cases**

$$P(0,b) = 1, P(a,0) = 0 (a > 0, b > 0)$$

**3: Establish the recurrence for the tabulation**

  $$P(a,b) = p \cdot P(a-1,b) + (1-p) \cdot P(a,b-1)$$

**4: Determine the order of subproblems**

Since $P(a,b)$ depends on $P(a-1, b)$ and $P(a, b-1)$, we need smaller values of a and b to be computed first, so we iterate a and b from 0 to k.

**5: Final form of output**

$$P(k, k)$$

**6: Put it all together as pseudocode**

```
SeriesWin(k, p):
	// Base cases
	for b = 1 to k:
		P[0, b] = 1
		
	for a = 1 to k:
		P[a, 0] = 0
	
	// Fill table
	for a = 1 to k:
		for b = 1 to k:
			P[a, b] = p * P[a-1, b] + (1-p) * P[a, b-1]
	return P[k, k]
```

**7: Runtime analysis**

There is a nested for loop with both a and b running from 1 to k, giving $O(k^2)$. Each iteration performs $O(1)$. So the total runtime is $O(k^2)$.

## Question 3

You are given a DAG $G$ such that each edge has positive edge length, a starting vertex $s$ and an ending vertex $t$. You are also given an integer $k$ such that $0\leq k \leq |V|$.

Design a DP tabulation algorithm that returns the length of the shortest path from $s$ to $t$ that uses exactly $k$ edges. (return $\infty$ if no such path exists).

**1: Define the subproblems:**

Let $D(v, j)$ be the length of the shortest path from s to t that uses exactly j edges.

**2: Define and evaluate the base cases**

$$D(s, 0) = 0$$
$$D(v, 0) = \infty$$

**3: Establish the recurrence for the tabulation.**

$$D(v, j) = \min_{(u, v) \in E} \{ D(u, j - 1) + w(u, v) \}$$

**4: Determine the order of subproblems:**

Since $D(v, j)$ depends on $D(u, j-1)$, we need smaller value of j to be computed first. So we iterate j from 0 to k.

5: Final form of output.

$$D(t, k)$$

**6: Put it all together as pseudocode**

```
ShortestPathKEdges(G, s, t, k):
	// Base cases
	D[s, 0] = 0
	for all v != s:
		D[v, 0] = ∞
	// Fill table
	for j = 1 to k:
		for each v in V:
			D[v, 0] = ∞
			for each incoming edge(u, v) in E:
				D[v, j] = min(D[v, j], D[u, j-1] + w(u, v))
	return D[t, k]
```

**7: Runtime analysis**

The outer loop runs k times. For each iteration, we go through all vertices and their incoming edges, which takes $O(|V|+|E|)$. So the total runtime is $O(K(|V|+|E|))$.
