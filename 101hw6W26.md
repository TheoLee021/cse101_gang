## Question 1
Let $G$ be an undirected graph with vertex set $V$ and edge set $E$. A *clique* is a complete subgraph $C \text{ of } G$. You wish to find the size of the largest clique $C \text{ of }G$ (the complete subgraph of $G$ with the most vertices.)

Consider the following backtracking algorithm that will achieve this:
```java
MaxClique(E):
	if |V| == 0:
		return 0
	Choose a vertex v in V
	Let G_v be the subgraph of G with vertex set V - {v} // in other words G_v can be constructed from G by deleting v and all edges connecting to v.
	OUT = MaxClique(G_v)
	Let Gp_v2 be the subgraph of G that contains all neighbors of v // and all edges connecting those vertices.
	IN = 1 + MacClique(Gp_v2)
	return max(IN,OUT)
```

**a)**. Prove the correctness of the algorithm.

For any undirected graph $G$ with $n$ vertices, the algorithm $MaxClique$ correctly returns the size of the maximum clique in $G$.

**Base Case**: when $|V| = 0$, MaxClique algorithm correctly return 0. Thus, the base case holds since .

**Inductive Hypothesis**: Let $n \gt 0$, MaxClique algorithm correctly returns the largest clique for any graph with $k$ vertices where $0 \leq k \lt n$.

**Inductive Step:** Prove that $MaxClique$ algorithm correctly returns the largest clique for $n$. Consider a arbitrary graph $G=(V,E)$ with exactly $n$ vertices. The algorithm will go to two cases to find the maximum clique $C$ of $G$ for the vertex $v \in V$.
- **Case 1:** Let the clique $C$ does **NOT** include $v$.  Result of $OUT$ will be the the maximum clique $C \text{ of } G$ by the inductive where the size of $C$ is exactly $n-1$.
- **Case 2:** Let the clique $C$ does contain vertex $v$ and all the neighbors of $v$. The result of $IN$ will be the maximum clique $C \text{ of } G$ by the inductive where the size of $C \leq (n-1)$.

**Conclusion:**, the $MaxClique$ algorithm correctly returns the largest clique for $n$ since the maximum clique of $G$ must either Case1 or Case2. The The algorithm returns $max(IN, OUT)$.

Therefore, by the Inductive, the algorithm is correctly return the maximum clique for all $n \geq 0$.

**b)**. Explain why the worst case runtime will give you a recurrence of 
$$
T(n)=2T(n-1)+O(n)
$$
where $n$ is the number of vertices.

There are 2 cases that the recurrent $MaxClique$ algorithm will be called in the $MaxClique$ algorithm.
- Case 1: the input of $G_{v}$ for the recurrence is exactly $n-1$ since it doesn't include itself only. Then, it is $T(n-1)$.
- Case 2: the input of $G'_{v}$ for the recurrence contains all neighbors of $v$ but doesn't cut the edges of $v$ from its neighbors. Therefore, the worst case for this recurrent is $v$ will contains all other vertices which the recurrence could be consider as $T(n-1)$ for this worst case.
Then, the algorithm of cutting out the subgraph might have the worst case that $v$ contains all others vertices. In other words, the edges $|E| = (n -1)$ for $v$. So the worst case of creating subgraph $G_{v}$ is $O(n-1)=O(n)$.

Therefore, the worst case runtime will be $T(n)=2T(n-1)+O(n)$ where $n$ is number of vertex.

**C).** Describe how you can improve this algorithm to get a worst case runtime of
$$
T(n)=T(n-1)+T(n-2)+O(n)
$$
where n is the number of vertices.

To implement this algorithm to approach $T(n)=T(n-1)+T(n-2)+O(n)$, we could check if the $G$ is a complete subgraph or not.
- Case 1, if $G$ is a complete subgraph, then we know it is a maximum $C$ with number $|V|$ vertices, then return $|V|$ which is considered as $n$. For checking it, the worst case is $O(n)$.
- Case 2, if $G$ is NOT a complete subgraph, we can guarantee that there will exist a $v$ such that it doesn't connect to all other vertices, where its $deg(v) \leq (n-2)$. Therefore we choose such a vertex $v \in V$, where $deg(v) \leq (n-2)$.
In conclusion, 
- The worst case for checking if the graph is complete subgraph or not is $O(n)$. 
- If it is not a complete subgraph:
	- The case $OUT$ still remain $T(n-1)$. 
	- The case $IN$ will runs $T(n-2)$ for its worst case since $deg(v) \leq (n-2)$ can be guaranteed. 

Therefore, 
$$
T(n)=T(n-1)+T(n-2)+O(n)
$$

## Question 2
Suppose you are playing a video game with a sequence of $n$ levels (starting at level 0.)

At the end of each level, there is a boss that will inflict $d_i$ amount of damage to your health. After you defeat the boss, you have the option to advance to the next level or skip forward $x_i$ levels.


You wish to know what is the least amount of damage inflicted in order to reach the end of the game (The start of level $n$.)

(The input is an array $((d_0,x_0),\dots,(d_{n-1},x_{n-1}))$.

(a) Design a reduction algorithm that uses Dijkstra's algorithm. Compute the time analysis in terms of the number of levels $n$.

Solution:

1. Build a graph

Create a graph with a vertex for each level 0, 1, ..., n

Create an edge from vertex i to vertex i+1 and to vertex $i+x_i$(if $i+x_i <= n$)

Weigh each edge with the damage $d_i$ of level i

2. Run algorithm

Run Dijkstra's algorithm on the graph starting at vertex 0

3. Output

Return the shortest path distance to vertex n

4. Runtime

The graph has n+1 vertices and at most 2n edges, so using a binary heap the runtime is O(nlogn)


(b) Design a DP tabulation algorithm by ordering the subproblems from $n$ to $0$: (step 1 and 4 have been done for you)

1: Define the subproblems:

Let $G[k]$ be defined to be the least amount of damage inflicted to get exactly to the end (start of level $n$) from level $k$.


2: Define and evaluate the base cases

Solution:

$$G[n] = 0$$

3: Establish the recurrence for the tabulation.

Solution:

$$G[k] = d_k + min(G[k+1], G[k + x_k])(if k + x_k <= n)$$

$$if\ k + x_k > n: G[k] = d_k + G[k+1]$$

4: Determine the order of subproblems:

Order the subproblems from $n$ down to $0$.

5: Final form of output.

Solution:

$$G[0]$$

6: Put it all together as pseudocode

Solution:

```
minDamage(d, x, n):
  G[n] = 0
  for k = n-1 down to 0:
    if k+$x_k$ > n:
      G[k] = d[k] + G[k+1]
    else:
      G[k] = d[k] + min(G[k+1], G[k+x[k]])
  return G[0]
```

7: Runtime analysis

The for loop iterates n time(from n-1 to 0) and all operations perform in each iteration are O(1), thus the runtime is O(n)

## Question 3

A contiguous subsequence of a sequence $S$ is a subsequence made up of consecutive entries of $S$.

Given a sequence of integers, design a DP tabulation algorithm that returns the sum of the maximum sum contiguous subsequence.

For example, if the input is $(5,15,-30,10,-5,40,10)$, then the maximum sum contiguous subsequence is $55 = 10 + (-5) + 40 + 10$ (the last 4 entries.) 

1: Define the subproblems:

Let M[k] be the maximum sum contiguous subsequence ending at index k.

2: Define and evaluate the base cases

M[0] = a[0] (input)

3: Establish the recurrence for the tabulation.

At index k, the best contiguous subsequence ending at k either extends the best subsequence ending at k-1 or starts fresh at k:

M[k] = max(a[k], M[k-1] + a[k])

4: Determine the order of subproblems:

Order the subproblems from 0 to n-1.

5: Final form of output.

max(M[0], M[1], ..., M[n-1])

6: Put it all together as pseudocode

```
M[0] = a[0]
best = M[0]
for k = 1 to n - 1:
  M[k] = max(a[k], M[k-1] + a[k])
  best = max(best, M[k])
return best
```

7: Runtime analysis

The for loop runs n-1 times and each iteration does O(1) work, so the runtime is O(n).

## Question 4

(12 points) You need to send a package but it will cost positive integer C cents. You open your drawer to find a collection of n individual stamps with positive integer values ($s_1,..., s_n$).

You wish to select a subset of the stamps so that the total cost of the stamps is exactly C and you wish to do this with the fewest number of stamps.

Design a DP tabulation algorithm that returns the minimum number of stamps required to add up exactly to C. (If it is impossible, then your algorithm can return ∞.)

**Solution:**

1: Define the subproblems:

Let L[k, c] be the minimum number of stamps used to make exactly c cents from a subset of the stamps $s_1,..., s_k$.

2: Define and evaluate the base cases

L[0,0] = 0 since there is one and one only way to make 0 cents using 0 stamps.

For (C $\geq$ 1), L[0,C] = $\infty$ because it is impossible to add up 0 stamps to a positive C.

3: Establish the recurrence for the tabulation.

Condition on whether we include stamp *k*th in our subset.

If we do not include stamp k: the minimum number of stamps that add up to C from $s_1,..., s_k$ is the same as the minimum number of stamps from $s_1,..., s_{k - 1}$

If we include stamp k: If $c - s_k \geq 0$, by using stamp of value $s_k$, we need to find the minimum of stamps that add up to $C - s_k$ cents from $s_1,...,s_{k - 1}$.

We take the minimum of both choices:
$$L[k, c] = min(L[k - 1, c], 1 + L[k - 1, c - s_k])$$

4: Determine the order of subproblems:

L[k,c] depends on both $L[k - 1, c]$ and  $L[k - 1, c - s_k]$, so we can order the subproblems as:

**for** k = 0,...,n:

and for each k, **for** c = 0,...,C:

and we will always have computed L[k - 1, c] and L[k - 1, c - $s_k$] before we need to compute L[k,c].

5: Final form of output.

L[n,C]

6: Put it all together as pseudocode

```java
FindMinStamp( {s_1,...,s_n}, C)
	for: i = 1,...,C
		L[0,C] = ∞

	L[0, 0] = 0

	for: k = 1,...,n:
		for: c = 0,...,C:
			L[k, c] = L[k - 1, c]
			if c - s_k >= 0 AND 1 + L[k - 1, c - s_k] < L[k, c]:
				L[k, c] = 1 + L[k - 1, c - s_k]

	return L[n, C]
```

7: Runtime analysis

The first loop runs in O(n) time. Then we have a nested loop, which the outer loop runs in O(n) times and the inner loop runs in C + 1 times. This nested loop runs in O(nC) time. Therefore, the algorithm runs in O(nC) time.
